
API là Application Programming Interface là một trung gian phần mềm cho phép hai ứng dụng giao tiếp với nhau. 
Nó là giao diện cho phép một ứng dụng giao tiếp với ứng dụng khác thông qua các lệnh đơn giản và cách các lệnh này được gửi và định dạng mà dữ liệu 
được truy xuất thông qua API có thể khác với API SOAP hoặc REST.
API có thể trả về dữ liệu  cho ứng dụng ở những kiểu dữ liệu phổ biến như JSON hay XML.

Đặc điểm nổi bật:
    -API sử dụng mã nguồn mở, dùng được với mọi client hỗ trợ XML, JSON.
    -API có khả năng đáp ứng đầy đủ các thành phần HTTP: URI, request/response headers, caching, versioning, content forma…. 
    -Mô hình web API dùng để hỗ trợ MVC. Ngoài ra, nó cũng hỗ trợ RESTful đầy đủ các phương thức như: GET, POST, PUT, DELETE các dữ liệu.
    -Được đánh giá là một trong những kiểu kiến trúc hỗ trợ tốt nhất với các thiết bị có lượng băng thông bị giới hạn như smartphone, tablet…


HTTP Request
    HTTP Request là yêu cầu được gửi từ máy khách (client) đến máy chủ (server)
HTTP Response
    HTTP Response là phản hồi từ máy chủ gửi lại cho máy khách. Nó bao gồm các thành phần chính sau:
Dữ liệu trong HTTP Request, HTTP Response:
    -Headers: Dữ liệu trong headers thường là dạng văn bản, ví dụ: Content-Type: application/json.
    -Body: Dữ liệu trong body có thể là nhiều loại khác nhau, chẳng hạn như văn bản, JSON, XML, 

Các HTTP methods chính:
    -GET: Lấy dữ liệu từ máy chủ. Không làm thay đổi trạng thái của máy chủ.
    -POST: Gửi dữ liệu tới máy chủ để tạo mới tài nguyên.
    -PUT: Cập nhật toàn bộ tài nguyên trên máy chủ.
    -PATCH: Cập nhật một phần tài nguyên trên máy chủ.
    -DELETE: Xóa tài nguyên trên máy chủ.



Phân biệt các phương thức
    -GET: Dùng khi cần lấy dữ liệu.
    -POST: Dùng khi cần tạo mới tài nguyên.
    -PUT: Dùng khi cần cập nhật toàn bộ tài nguyên.
    -PATCH: Dùng khi cần cập nhật một phần tài nguyên.
    -DELETE: Dùng khi cần xóa tài nguyên.



So sánh sự khác nhau giữa các methods: PUT, PATCH, POST
    PUT vs PATCH
PUT: Cập nhật toàn bộ tài nguyên, nếu tài nguyên không tồn tại thì tạo mới.
PATCH: Chỉ cập nhật một phần tài nguyên.
    POST vs PUT
POST: Tạo mới tài nguyên, mỗi lần gọi sẽ tạo ra một tài nguyên mới.
PUT: Cập nhật hoặc tạo mới tài nguyên tại một URI cụ thể.
